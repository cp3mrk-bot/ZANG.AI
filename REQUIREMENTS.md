# ZANG爱 锐评社区 - 产品需求文档

## 一、产品概述

### 1.1 产品定位
- **产品名称**：ZANG爱
- **产品定位**：锐评/吐槽社区 H5 应用
- **目标用户**：喜欢在互联网上冲浪、吐槽、锐评的年轻用户群体
- **核心价值**：用最犀利的眼光，锐评一切值得锐评的事物

### 1.2 品牌形象
- **Logo风格**：葬爱杀马特风格（长发烫头男头像）
- **AI助手名称**：小ZANG
- **AI人设**：喜欢在网上冲浪、对互联网热梗有深刻认知的 AI，锐评风格参考葬AI、X博士、半佛仙人

---

## 二、功能需求

### 2.1 信息流（主页）
- **布局风格**：模仿早期推特的信息流设计
- **内容展示**：
  - 用户头像 + 昵称 + 发布时间
  - 帖子内容（文字 + 图片）
  - 互动按钮（评论、点赞、转发）
  - 标签分类展示

### 2.2 发帖功能
- **文字限制**：300字以内
- **超长处理**：超过300字自动转为文章链接形式（类似公众号文章链接）
- **支持内容**：
  - 纯文字锐评
  - 上传图片（支持多图）
  - AI生成图片
  - 让AI帮忙锐评

### 2.3 AI锐评功能
- **触发方式**：
  1. 发帖时点击"帮我锐评"
  2. 在帖子或评论区 @小ZANG
- **AI风格**：
  - 葬AI风格：犀利、毒舌、一针见血
  - X博士风格：深度分析、另类视角
  - 半佛仙人风格：商业视角、反讽幽默
- **回复形式**：流式输出，打字机效果

### 2.4 评论功能
- 支持对帖子进行评论
- 支持在评论区 @小ZANG 让AI进行锐评
- 评论支持文字和图片

### 2.5 内容分类
- AI应用产品
- 数码产品
- 明星八卦
- 职场
- 家庭关系
- 其他（用户自定义标签）

---

## 三、技术方案

### 3.1 技术栈
- **前端**：Next.js 16 + React 19 + TypeScript + Tailwind CSS
- **UI组件**：shadcn/ui
- **后端**：Next.js API Routes
- **数据库**：Supabase（PostgreSQL）
- **AI能力**：
  - 文本生成：豆包/DeepSeek（用于小ZANG锐评）
  - 图像生成：AI图像生成（用于用户生成图片、Logo）
- **存储**：对象存储（用户上传的图片）

### 3.2 核心API
- `/api/posts` - 帖子 CRUD
- `/api/posts/[id]/comments` - 评论管理
- `/api/ai/roast` - AI锐评接口
- `/api/ai/generate-image` - AI生成图片
- `/api/upload` - 图片上传

### 3.3 数据模型
- **User**: id, nickname, avatar, created_at
- **Post**: id, user_id, content, images, category, is_article, article_url, likes, created_at
- **Comment**: id, post_id, user_id, content, images, created_at
- **AIRoast**: id, post_id/comment_id, content, created_at

---

## 四、UI/UX 设计

### 4.1 主题风格
- **色彩**：复古、大胆、有个性（避免蓝紫色渐变）
- **字体**：有个性、非主流感
- **整体调性**：杀马特 + 赛博朋克 + 复古互联网

### 4.2 页面结构
1. **首页**：信息流 + 发帖入口
2. **发帖页**：文字输入 + 图片上传/生成 + AI锐评
3. **帖子详情页**：帖子内容 + 评论区
4. **个人中心**：我的帖子 + 我的评论
